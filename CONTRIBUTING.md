# Contributing

Two ways to add an integration:

1. **Issue form (recommended)** — open [Add an integration](../../issues/new?template=integration-submission.yml). No JSON knowledge needed.
2. **Pull request against `/data`** — for contributors comfortable with the schema.

## Rules

- **Only ever edit `/data`** (the `*.json` category files plus the `labels.json` and `services.json` registries). Never edit `README.md` — it is generated by `scripts/generate.mjs`.
- The **filename is the category** — there is no `category` field.
- One listing per artifact. Every field in `schema.json` is required.
- `services` is an array of ids from `services.json` — reuse an existing id if the service is already listed; add a registry entry (id + display `name`) if it isn't.
- CI validates `/data` against `schema.json` and rejects duplicate `id`s, dangling enum tokens, and services ids missing from `services.json`.

Submissions are reviewed manually. No accounts, no self-service listings.
